# 高校公众号运营平台 · 提示词系统说明文档

> 本文档面向业务人员，无需技术背景即可读懂。
> 目标：让你清楚地知道——用户点了什么按钮、AI 收到了什么指令、这条指令是怎么拼出来的。

---

## 一、核心思路：AI 收到的是一份完整的"作战简报"

每当用户触发一次 AI 生成，系统会在后台自动组装一份结构化的简报发给 AI。这份简报包含三个部分：

```
提示词 = ① 学校数据块 + ② 任务描述 + ③ HMS规范底座
```

| 部分 | 内容 | 来源 |
|------|------|------|
| ① 学校数据块 | 学校定位、宣传重点/禁区、校园标志、招生数据、王牌专业 | `data/school-profiles.ts`，用户在导航栏选择学校后自动注入 |
| ② 任务描述 | 本次具体任务（热点/节点/搜索词/参考文章） | 根据用户点击的入口不同而变化 |
| ③ HMS规范底座 | HMS三维模型 + 五类选题矩阵 + 标题要求 + 创作红线 | `lib/prompt-builder.ts` 中的 `HMS_FRAMEWORK` 常量，所有入口共用 |

---

## 二、两条生成路径

### 路径一：有配置面板（四维精准选题）

**触发方式**：在搜索框下拉面板里选好学校、节点、文章类型、写手风格四项，点"AI 选题"。

**AI 收到什么**：

```
① 学校数据块（完整版：定位 + 宣传重点 + 禁区 + 招生数据 + 王牌专业）
② 节点信息（节点标题、时间、说明、可策划方向、特别提示）
   + 写手风格描述（5种之一，详细约束语言和句式）
   + 节点语气约束（如有，例：清明节必须庄重）
   + 节点强绑定要求（所有选题必须以该节点为核心，不得偏题）
③ HMS规范底座
```

按节点类型路由到三套专用提示词：

| 节点类型 | 函数 | 受众 | 特殊规则 |
|------|------|------|------|
| 招生节点 | `buildRecruitPrompt` | 高考生、家长 | 不涉及考研/保研 |
| 节气/节日 | `buildFestivalPrompt` | 在校生、校友、社会大众 | 受节点语气约束 |
| 校园节点 | `buildCampusPrompt` | 在校生、新生、校友 | 强调归属感 |

---

### 路径二：直接触发（快速探索，无需配置）

**触发方式**：直接点击节点卡片、输入搜索词、点热点、洞察场域参考他校。

**AI 收到什么**：

```
① 学校数据块（简化版：定位 + 宣传重点/禁区 + 校园标志）
② 任务描述（根据入口不同）
③ HMS规范底座（与路径一完全相同）
```

| 入口 | 函数 | ② 任务描述内容 |
|------|------|------|
| 节点卡片直接点击 | `buildNodeQuickPrompt` | 节点标题、类型、说明、可策划方向、语气约束（如有） |
| 搜索框自由文本 | `buildFreeSearchPrompt` | 用户输入的关键词 |
| 全网热点 | `buildHotTopicPrompt` | 热点标题、来源、标签、热度 |
| 洞察场域参考他校 | `buildInsightReferencePrompt` | 参考学校名、参考文章标题 |

路径二比路径一少了：写手风格、节点强绑定要求。适合快速探索，不需要精细配置。

## 三、HMS规范底座（所有入口共用）

这是所有提示词的"行为准则"，保证每次生成都遵循同一套方法论。

### 3.1 HMS 三维驱动模型

每个选题必须有且仅有一个核心驱动维度：

| 维度 | 全称 | 目标 | 成功标准 |
|------|------|------|------|
| **H** | Heart · 动心 | 连接情感，触发认同或共鸣 | 读者觉得"这就是我/我们" |
| **M** | Mind · 有用 | 解决问题，提供可操作信息 | 读者忍不住收藏或转发 |
| **S** | Smile · 会心一笑 | 创造愉悦，用幽默解构严肃 | 读者笑着分享，评论"是我校了" |

**重要原则**：
- 拒绝拼盘——一个选题不能同时追求"既感人又实用又好笑"
- 拒绝套路——严禁套用固定句式模板
- 逻辑自洽——情感升华必须建立在合理逻辑上

### 3.2 五类选题矩阵

| 类型 | 核心任务 |
|------|------|
| 新闻资讯软化类 | 为冰冷的"事"找到温暖的"人" |
| 招生宣传种草类 | 贩卖"体验"和"可能性"，不罗列排名 |
| 热点借势融合类 | 把社会情绪"翻译"成校园语境 |
| 实用指南痛点类 | 做读者在特定阶段的"救命锦囊" |
| 校园故事共情类 | 发现并礼赞"平凡中的光辉" |

### 3.3 标题创作要求

标题必须**长短错落、句式多变**：

- 短标题（6-10字）：一句话点破核心，如"她在图书馆哭了一整晚"
- 中标题（11-16字）：有情境感，如"毕业前我终于去了一次随园深处的那栋楼"
- 疑问/祈使句：制造对话感，如"你真的了解自己选的这个专业吗"
- 口语化短句：像朋友说话，如"今年的银杏比去年更黄了"
- **严禁**：所有标题都用"xxx，xxx"或"xxx：xxx"的切割句式

## 四、学校档案

每所接入的学校，产品里存有一份标准化档案（`data/school-profiles.ts`）：

| 字段 | 内容示例（南师大） | 作用 |
|------|------|------|
| 学校定位 | 百年师范名校，人文底蕴深厚的综合性研究型大学 | 让 AI 建立整体认知 |
| 宣传重点 | ①百年人文底蕴与随园美学 ②教师教育品牌… | AI 生成内容时优先这些方向 |
| 宣传禁区 | 不要过度强调就业薪资；不要和985硬比排名 | 避免踩雷 |
| 校园标志性元素 | 随园、北大楼、银杏大道、随园食堂 | 让选题有专属细节感 |
| 招生数据 | 在校约2.4万人；录取线高出一本线20-40分 | 写招生内容时可引用 |
| 王牌专业 | 教育学（全国师范类第一）；数学（强基计划核心） | 专业类内容时注入 |
| 就业亮点 | 高中教师"南师大制造"遍布全国重点中学 | 就业方向类内容 |
| 推荐写作风格 | 温柔人文风 | 与学校气质匹配 |
| 节点特别提示 | 9月10日校庆日与教师节同日，全年最独特情感节点 | 特殊节点专属内容提示 |
| 不适用节点 | 强基计划（211院校无此政策） | 过滤掉无意义的节点 |

---

## 五、五种写手风格（路径一专有）

| 风格 | 适合学校类型 | AI 的行为约束 |
|------|------|------|
| 温柔人文风 | 师范、文科类（如南师大） | 细腻温暖、多短句疑问句；禁用"欣慰""蓬勃发展"等官话 |
| 严谨工科风 | 工科类（如东南大学） | 数据说话、逻辑清晰；禁止过度煽情 |
| 专业权威风 | 医科类（如南医大） | 专业严谨但有人文关怀；不夸大薪资，不回避培养年限 |
| 活泼校媒风 | 民办本科（如金科） | 年轻化网感、多换行；禁说官方话 |
| 权威官方风 | 顶尖名校 | 严肃大气、从国家战略高度切入；禁止娱乐化 |

---

## 六、节点语气约束（特殊节日专有）

| 节点 | 约束内容 |
|------|------|
| 清明节 | 必须庄重肃穆；踏青内容语气宜淡雅；严禁活泼语气和段子标题 |
| 国家公祭日 | 必须高度庄重；严禁任何轻松娱乐化内容 |
| 元旦 | 可喜庆充满希望，但避免过度狂欢，宜有格调 |
| 五一劳动节 | 可轻松活泼，但需有一定内容厚度 |
| 南师大校庆日·教师节 | 双节叠加；温暖有历史感；兼顾对教师的致敬和对校园的眷恋 |

---

## 七、节点过滤规则

日历里的节点不是对所有学校一视同仁显示：

| 过滤规则 | 例子 |
|------|------|
| **985专属节点** | 强基计划招生简章——仅985院校可见；211/普本/民办均不显示 |
| **学校专属节点** | 校庆日·教师节双节叠加——仅南师大可见 |
| **通用节点** | 高考倒计时、清明节、五一劳动节——所有学校均可见 |

---

## 八、如何新增一所学校

1. **在 `data/school-profiles.ts` 里添加该校档案**
   - 填写学校名称、层次、特色、定位
   - 填写宣传重点/禁区、校园标志、招生数据、王牌专业
   - 填写推荐写作风格、不适用节点列表
   - 如有特殊节点（如校庆日），填写 `specialNodeHints`

2. **（可选）在 `data/njnu-calendar.ts` 里给某节点加上 `schoolOnly: '该校名称'`**

完成后，该学校自动出现在导航栏选择器里，所有 AI 入口自动注入该校档案。
